# Plan for Implementing Story: sa-01

Story: `artifacts/stories/sa-01-home-page-tool-menu.md`

## Tasks
1. sa-01-t1
  - Description: Add `--card` and `--card-border` theme tokens (light + dark) to `src/app/globals.css`.
  - Acceptance Criteria:
    - Tokens defined in `:root`, dark values in `prefers-color-scheme: dark` block.
    - Exposed via `@theme inline` as `--color-card` / `--color-card-border`.
  - Status: Completed

2. sa-01-t2
  - Description: Create stub Intersect page at `src/app/steam-intersect/page.tsx`.
  - Acceptance Criteria:
    - Route `/steam-intersect` renders title "Intersect" + "coming soon" note.
    - Uses shared theme tokens, no one-off colors.
  - Status: Completed

3. sa-01-t3
  - Description: Rebuild home page (`src/app/page.tsx`) as tool menu.
  - Acceptance Criteria:
    - "Arsenal" heading + tagline above grid.
    - Static inline `tools` array of `{ name, description, path }`, one entry: Intersect.
    - Responsive card grid (1 col mobile, 2-3 desktop); whole card clickable `next/link` to `/intersect`.
    - Cards styled with `bg-card` / `border-card-border` tokens.
  - Status: Completed

4. sa-01-t4
  - Description: Update metadata in `src/app/layout.tsx`.
  - Acceptance Criteria:
    - Title "Arsenal", description reflects multi-tool site.
  - Status: Completed

5. sa-01-t5
  - Description: Verify story: `npm run build`, `npm run lint`, manual run of dev server checking desktop + mobile widths.
  - Acceptance Criteria:
    - Build and lint pass.
    - Home page shows Intersect card; clicking navigates to `/steam-intersect`.
  - Status: Completed

## Excludes
- Intersect tool functionality (sa-02, sa-03).
- Unit tests / test runner setup (backend-only policy; next story).
- Shared header/nav component.
- `lib/` or `components/` extraction — tool list stays inline until reuse needed.
